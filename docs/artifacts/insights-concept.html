The $1.18B headline overstates urgency — $540M of it is Residential under 36 months, which re-appraises cheaply and rarely moves value. The genuinely exposed slice is the Office >48-month corner ($310M), where cap rates have moved most since. Prioritise that corner, not the raw count.

1 · ClaimA single, falsifiable headline with the number and the dollar impact. No hedging in the title — the hedging lives in the read.
2 · EvidenceThe smallest chart that proves the claim (here, the class × age heatmap). It's proof, not exploration — the dashboard is where you browse.
3 · AI readThe differentiator. Bricklayer critiques its own headline — narrows $1.18B down to the $310M that actually matters. Skeptical by default.
4 · ActionA concrete next step (queue for re-appraisal) plus Dismiss — and dismissals are how the opinion model learns per bank (v2).
✓ Verified + provenanceThe badge means the numbers reconciled against the Data Lake; the mono provenance line names the exact source object. This is "receipts."

Condition drives price cleanly from Excellent ($268) down to Fair ($96) — except "Poor" prices above Average, which is nonsensical. That $191 is almost certainly a definitional or data-entry artifact — likely a handful of mislabelled records — rather than a true condition-driven pricing signal. 08
Insights ≠ Dashboard

The opinion layer, not another grid of charts.

This resolves the July-14 duplication concern head-on. Insights and the dashboard use the same data lake but do different jobs — one argues, one browses.

Insights — the opinion / critique layer

  • Proactive — it raises what's worth a look; you don't go hunting.
  • Every item runs claim → evidence → AI read → action.
  • Charts are evidence — the smallest proof of a claim, then critiqued.
  • Per-lens landing; it's where you arrive.
  • Skeptical: it challenges its own numbers.

Dashboard — the browse layer

  • Reactive — a saved collection of report tiles you explore.
  • Tiles answer "what is X?"; you drill, filter, rearrange.
  • Charts are interactive tiles — the destination, not proof.
  • A default dashboard per lens; you navigate to it.
  • Neutral: it shows the number, it doesn't argue about it.

The bridge: a lighter pinned insight card can sit atop a dashboard

For the in-context case, one finding — claim + read + a "See full insight →" link back to the Insights landing — can pin to the top of a dashboard. It borrows the pattern without duplicating the tiles below it.
