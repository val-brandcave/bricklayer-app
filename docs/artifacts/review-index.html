The course correction · July 15

What changed

The July 14 discovery built a bespoke IA. On the call, Cody redirected us onto HubSpot's skeleton. We audited the rebase against all 25 of Ed's POC screenshots — it does not lose him; in fact he said "an entire new dashboard," and Cody said "very HubSpot" twice during Ed's own demo.

Was — July 14 bespoke IANow — Direction v2 (HubSpot-anchored)
"Boards"→Dashboards + Reports — HubSpot nouns, verbatim
Briefing / Boards / Properties / Deliverables→HubSpot object model + Insights landing + Chat; Deliverables → actions
One undifferentiated chat→Two surfaces — right-docked co-working + full-page threaded
~12 bespoke primitives→14 widget types × two forms (dashboard tile + in-chat MCP app)
Insights ≈ a second dashboard→Insights = the opinion / critique layer, per-lens, never a dashboard copy
Decisions · locked July 20

What's decided

✓Naming — Dashboards + Reports ("board" retired)
✓Chat dock — right (co-working) + full-page as its own destination
✓Insights — a real per-lens landing; opinion model fixed → tuned → learns
✓Widget set — 9 + scatter, map, heatmap, watchlist, scenario + starter reports
✓Both forms — the same component, chat-framed with Save / Edit
✓Create/edit — app-level modals, not the in-chat drawer
✓Deliverables — actions on dashboards & reports, not a tab
✓RBAC lens — set at provisioning, no switcher (unchanged)
Still open Eval surfacing (badges + provenance now vs. a visible review queue later) and the per-lens default dashboard content — deliberately deferred until Ed's video + the Friday bank demo, per Cody's own framing. Neither blocks the package.
